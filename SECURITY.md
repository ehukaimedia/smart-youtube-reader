# Security Policy

## Supported Versions

We actively support and patch security issues for the following versions of Smart YouTube Reader:

| Version | Supported |
| ------- | --------- |
| 0.x     | :white_check_mark: Supported / Current Release |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please do **not** open a public issue. Instead, report it directly to the maintainers at Ehukai Media.

### Contact Information
Please email security reports to: **security@ehukaimedia.com**

Please include in your report:
* A detailed description of the vulnerability.
* A proof of concept (PoC) or step-by-step instructions to reproduce the issue.
* The potential impact (e.g., local file disclosure, arbitrary command execution).
* Any suggested mitigations or fixes.

### Our Response Process
We take security reports seriously. Once a vulnerability is reported:
1. We will acknowledge receipt of your report within 48 hours.
2. We will investigate the issue and coordinate a fix.
3. We will keep you updated on our progress.
4. We aim to release a patched version within 30 days of the report, depending on complexity.
5. Credit will be given to the reporter in our release notes unless anonymity is requested.
