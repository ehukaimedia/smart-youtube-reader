HF Gemma 4 QAT Model Evaluation

Intent

This playground informs one decision: does the Hugging Face QAT release hf.co/google/gemma-4-12B-it-qat-q4_0-gguf replace gemma4:12b as the Smart YouTube Reader default model? It exists to prevent a specific anti-pattern: swapping the production model because a release is described as "faster and more accurate" elsewhere, without measuring it on this app's own chaptering and vision workload, on the runtime and hardware the app actually uses. The full protocol and verdict live in docs/specs/hf-gemma4-qat-model-evaluation.md; raw logs in docs/benchmarks/gemma4-12b-qat-vs-q4km-2026-06-10/.

Verdict — NO-GO (2026-06-10)

Keep gemma4:12b. Across 54 identical text calls and 22 vision selections per model on an RTX 5070 / Ollama 0.30.7: the QAT candidate passed 20.4% of quality gates vs the baseline's 29.6%, was not faster (10.77 vs 10.80 s/call overall; +10.5% slower on the production schema-JSON path), produced more malformed outputs (14 vs 10), over-segmented chapters in all three schema runs on the TED-Ed job (6–7 chapters where the contract allows 2–5), and fell back to deterministic image selection once (baseline: never). Its only measured advantage is ~400 MB less resident VRAM (7.7 vs 8.1 GB).

Spec Seed

Decision: Keep gemma4:12b (Ollama library, Q4_K_M) as DEFAULT_MODEL. Reject hf.co/google/gemma-4-12B-it-qat-q4_0-gguf as default: not faster, less accurate on the production schema-JSON path, systematic chapter-count over-segmentation, one vision fallback.

Standing gate: a future candidate replaces the default only with: schema_json pass rate ≥ baseline + 10 pp (or equal pass rate with ≥ 20% lower latency); no overall pass-rate or error regression; vision method rate ≥ baseline; artifact digest-pinned and license-compatible; committed evidence under docs/benchmarks/.

Ownership: backend/app/model_runtime.py owns model identity and the chat boundary; the dormant migration recipe (6 touch points) is in docs/plans/hf-gemma4-qat-evaluation-and-model-gate-2026-06-10.md.

Caveats recorded: sampling params not fully matched (library modelfile sets top_k=64, top_p=0.95; HF import uses runtime defaults) at temperature 0.1; single machine; candidate GGUF shows irregular stop tokens suggesting early template conversion.

Non-goals: no runtime replacement, no new model-serving dependencies, no removal of deterministic fallbacks, no second 12B-class catalog entry that loses on the primary path.

Re-evaluation triggers: Google QAT GGUF re-upload or an Ollama library -it-qat tag; material Q4_0 vs Q4_K_M kernel changes; chaptering contract changes.
