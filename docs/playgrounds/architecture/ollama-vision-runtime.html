Ollama Vision Runtime Architecture

Intent

Smart YouTube Reader has one local model path: regular Ollama Gemma 4, defaulting to gemma4:12b. It uses text for transcript chaptering and image input for chapter frame selection. This playground prevents drift back to retired providers, text-only model tags, or Apple-only setup claims.

Runtime Flow

PipelineDownloads video, extracts transcript text, slices frames, hashes and scores frame metadata.
Ollama Runtimemodel_runtime.chat() calls /api/chat and preserves attached images.
ChapteringTranscript chunks go to Gemma 4 as text and return XML/JSON chapter structure.
Vision SelectionA bounded candidate frame set goes to Gemma 4 vision; only returned candidate filenames are accepted.
pipeline.py -> intelligence.py: create_ai_archive() receives transcript and FrameManager.
intelligence.py -> model_runtime.py: _chat() is the only model call boundary.
/models -> new project UI: only installed Ollama tags are selectable; catalog-only tags stay metadata.
FrameManager -> vision selector: frame metadata narrows candidates before images are encoded.
Vision selector -> archive.json: writes images, _image_context, and _image_selection.

Spec Seed

Decision: Use Ollama gemma4:12b for transcript chaptering and vision-assisted image selection. Deterministic image scoring remains only as a fallback when vision selection fails.

Model readiness: /models returns only installed tags in models, while model_details keeps uninstalled catalog entries informational. /jobs checks the requested model before creating a background job.

Non-goals: No raw audio ingestion, no cloud requirement for initial archive generation, no text-only Gemma 4 local-model tags, and no provider abstraction that preserves an obsolete runtime.

Acceptance: backend tests and ruff pass; model benchmark prints Ollama runtime; active docs and code have no retired runtime claims; uninstalled catalog models are not normal selectable choices; real project image benchmark reports ollama_vision or an explicit deterministic fallback reason.

Source Anchors

Runtime: backend/app/model_runtime.py owns Ollama host, default model, model checks, and image-preserving chat payloads.
Model API: backend/app/main.py::list_models separates installed selectable models from catalog metadata.
Chaptering: backend/app/intelligence.py::_generate_archive_chunk sends transcript text through the runtime boundary.
Vision: backend/app/intelligence.py::_choose_vision_representative_frames sends candidate frames to Gemma 4 and validates returned filenames.
Fallback: backend/app/intelligence.py::_choose_representative_frames remains deterministic recovery behavior.
